# Learnings — 2026-07-12

## BashTool crash on malformed LLM input
**Domain(s):** BashTool, ToolInput, safety
**Approach:** The BashTool crashed when the LLM sent `{"cmd": "..."}` instead of `{"command": "..."}` because `isReadOnlyCommand(undefined)` called `undefined.split()`. Fix: guard both `isReadOnlyCommand` and `isDangerousCommand` against falsy input, and add `normalizeInput()` to map `cmd` → `command`.
**Outcome:** BashTool now safely normalizes LLM tool call inputs, preventing crashes from wrong field names.
**Lesson:** Always guard LLM tool call inputs against wrong field names — the LLM may send any schema. This is a tool input robustness pattern: normalize malformed input at the boundary, guard every function that touches potentially undefined values. Apply this pattern to all tools that receive LLM-generated arguments.
